# TryHackMe - Phishing Simulation Investigation

## Overview

This project documents a phishing investigation completed as part of a TryHackMe phishing simulation challenge.

The objective was to act as a SOC Analyst and investigate multiple security alerts within a limited timeframe. Using Splunk, I analysed logs and indicators to determine whether each alert represented a true positive or a false positive. The challenge focused on alert triage, log analysis, and decision-making under pressure, which are key responsibilities in a Security Operations Centre (SOC).

## Scenario

A series of phishing-related alerts were generated within the environment. As the analyst, my task was to investigate each alert, review the available evidence, and determine whether malicious activity had occurred.

The investigation involved working against the clock while managing four separate alerts and prioritising findings efficiently.

## Objectives

- Investigate phishing-related security alerts.
- Analyse logs using Splunk.
- Identify indicators of compromise (IOCs).
- Determine whether alerts were true positives or false positives.
- Improve incident triage and investigation skills.
- Gain hands-on SOC analyst experience.

## Technologies Used

- TryHackMe
- Splunk Enterprise
- Security Alert Data
- Browser and Email Artifacts
